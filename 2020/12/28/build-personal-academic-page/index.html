如何快速搭建个人学术页面Jekyll Academic + Github Pages

本文受到个人网站搭建的启发

首先你需要有：

  • 本地git
  • GitHub账号

使用本方法可以避免本地hexo、hugo安装过程中遇到的各种问题(有些甚至找不到缘由或者没有人给出解决方案),此方法不限于创建学术界面，创建博客等一样适用。基本思路：

  1. 使用GitHub的fork功能直接“复制”你喜欢的GitHub模板到本地
  2. 用Git Bash Here在本地创建git仓库
  3. 修改配置文件(一般是源目录下的yml文件,可以参考该模板的官方说明)
  4. 写博客(可选)
  5. 用Git Bash Here提交刚才的修改

具体方法

普通流程

  1. Fork该仓库.
  2. 回到自己的GitHub页面，在该仓库Setting中修改仓库名称为[your_name].github.io，该名称将会成为你个人主页的网址。修改完成后直接访问[your_name].github.io就可以看到页面搭建完成(有时需要等两分钟配置完成)。

关键分支：在GitHub上直接修改

优点：操作直接，理解容易
缺点：增加文件很麻烦，没办法删除文件(致命缺点)

  1. 设置侧边内容、生成content&metadata。
    • Site-wide configuration : 主要的配置信息及侧边信息在_config.yml中，top menu的配置在_data/navigation.yml中，对于不需要的选项可以删除(注释掉)。
    • Create contenet & metadata : 网站内容都存储在对应文件夹(如_publications, posts等)下的markdown文件中，这些markdown文件的顶部是yaml格式来定义结构化内容。
    • 根据该主题的Guide，可以根据关键文件的位置进行内容修改，直接修改md文件即可不必安装Jekyll。很多教程中的第一步就是安装Jekyll，其实没有必要安装该框架，因为Github提供了Jekyll的服务环境，会自动利用Jekell生成站点，用户只需要修改关注md文件即可。
    • 关键文件位置及路径
      Basic config options: _config.yml
      Top navigation bar config: _data/navigation.yml
      Single pages: _pages/
      Collections of pages are .md or .html files in:
      _publications/
      _portfolio/
      _posts/
      _teaching/
      _talks/
      Footer: _includes/footer.html
      Static files (like PDFs): /files/
      Profile image (can set in _config.yml): images/profile.png
  2. 上传文件（pdf, zip等文件）至files/directory下，会出现在https://[your name].github.io下。
    在设置页面修改完善”GitHub pages”页面
  3. 目前搭建好的个人主页可以直接访问[your_name].github.io，如我的个人主页zyx97.github.io。但如果想要使用独有的个性域名还需要额外步骤。

关键分支：在本地修改

本段受到如何修改别人的github项目，并上传到他的github的启发
优点：增删查改更符合习惯
缺点：要用git，上传时输密码很麻烦(待解决)

  1. 在文件夹里打开Git Bash Here
  2. 输入git init创建本地仓库（结果是在当前目录会出现.git文件，为隐藏文件夹）
  3. 输入git clone https://github.com/[your_name]/[your_name].github.io 你的github项目地址 （这一步是克隆项目到本地），此时会出现[your_name].github.io的文件夹，点进去发现内容与你的GitHub一致
  4. 同上一个关键分支，直接修改各个配置文件
  5. (可选)阅读该模板的Guide，用Git或者手动创建新的博客(.md文件)，用自己喜欢的编辑器写博客
  6. 在[your_name].github.io文件夹内打开Git Bash Here
  7. 输入 git add .（这一步是将修改的内容进行同步，须注意的是后面的 . 要有空格）在这个输入完后可以在输入git status查看进度
  8. 输入git commit -m "你的修改地方" （这一步可以跳过，主要作用是对修改的地方进行注释，方便整合管理），此时Bash会显示你做了哪些修改(例如删掉了什么文件)
  9. 输入 git push即可上传成功，此处需要账号密码登录GitHub(不然岂不是谁都可以修改你的文件并覆盖)，且GitHub给的登录窗口永远登陆失败，然后Git会再给你一个SSH登陆窗口，再次输入账号密码，当看到几个100%和done就表示成功了,此时去GitHub看看自己库有没有相应的修改
  10. 访问[your_name].github.io看看自己的博客

使用自己的网址

  1. 在自己购买域名的网站(阿里云,name.com等)设置一条DNS解析，类型为CNAME,主机为自己拥有的域名或者子域名，响应为[your_name].github.io，其余默认即可
  2. 在GitHub库的settings中下滑到GitHub Pages，找到Custom domain，填入刚才的主机名并保存
  3. 过几分钟(或更久)访问自己的域名就会自动跳转到博客
